- 기본 규칙
	- 검색 조건에 일자(기간)은 default 1주일로 한다 -> 공통 규칙
	- 추가,보기,수정 화면은 1개 화면으로 구성
    - 수정 화면에서 수정 불가 항목도 표시는 해줘
	- 모든 삭제는 Database에서 삭제하지 않고, 삭제 Flag관리한다.
	- 모든 목록 조회는 삭제 Flag가 N인 것만 보여준다.
    - 등록, 수정, 삭제가 발생하면 수정일자, 수정자 정보를 업데이트
    - 모든 주소 검색은 Daum 주소 검색을 연동시켜주고 우편번호, 상세 주소 입력란을 만들어줘
    - 모든 목록은 Page Navigation을 적용해줘
    - DB Table도 직접 생성해줘
    - 등록, 수정, 보기 화면등은 Layer Popup 형태로 띄워줘
    - 등록, 수정, 삭제가 이뤄진 후에는 목록을 갱신해줘
    - 날짜 기간 검색시 시작은 해당일 0시0분0초 부터, 종료는 해당일 11시 59분59초까지로 해줘
    - 모든 검색 조건 화면에는 검색 버튼 이외 초기화 버튼을 추가해서 클릭시 검색 조건 초기화가 되도록 해줘
    - email 발송은 github에 있는 ig-notification와 연동하도록 하며, https://ig-notification.ig-pilot.com 을 사용
    - SMTP 설정은 aws secret manager에 "prod/ignite-pilot/smtp-naver" 값을 참고해서 발송해줘
    - 서비스에서 제공하는 삭제 기능은 해당 데이터가 사용되지 않을 경우만 삭제를 진행해주고, 사용되고 있어서 삭제가 불가능한 경우. 어디서 사용되고 있어서 삭제가 안되는지 Alert Popup에서 안내해주도록 해줘. (최대 3개까지)
    - 파일 첨부는 모두 AWS S3에 등록을 해줘
