# 도메인 개념 정의

## 작업 (Work)

**작업은 공정의 가장 작은 단위입니다.**

하나의 작업은 공장 바닥에 위치한 물리적 설비(Station, Buffer, Conveyor)를 실제로 사용하는 행위를 나타냅니다.

- 작업자가 어느 **스테이션(Station)**에서 가공을 수행하거나
- 부품을 **버퍼(Buffer)**에 대기시키거나
- **컨베이어(Conveyor)**를 통해 이동시키는 것이 각각 하나의 작업입니다.

### 작업과 도면 심볼의 연결

공간 관리(Plans) 메뉴에서 도면을 편집할 때, **STATION / BUFFER / CONVEYOR** 분류의 심볼에 작업을 연결할 수 있습니다.

이 연결 정보는 `plan_symbol_overrides.work_id`에 저장됩니다.

```
도면 심볼 (plan_symbol_overrides)
  └─ work_id → work.id  (어떤 작업에 사용되는 설비인지)
```

## 공정 (Process)

**공정은 여러 작업(Work)의 시퀀스입니다.**

하나의 제품 또는 반제품을 완성하기 위해 여러 Station을 거치는 일련의 흐름 전체를 공정이라 부릅니다.

```
공정 (Process)
  └─ 공정 단계 (ProcessStep) × N
       └─ 작업 (Work)         ← 최소 단위
            └─ 설비 사용 (Station / Buffer / Conveyor)
```

### 예시

| 단계 | 작업명       | 사용 설비      |
|------|-------------|---------------|
| 1    | 원자재 투입  | Conveyor-A     |
| 2    | 1차 가공     | Station-01     |
| 3    | 중간 버퍼링  | Buffer-B       |
| 4    | 2차 가공     | Station-02     |
| 5    | 완제품 배출  | Conveyor-C     |

---

## 시뮬레이션과의 관계

시뮬레이션(Simulations) 메뉴의 컴포넌트 타입(SOURCE, DRAIN, STORAGE, CONVEYOR, STATION)은 공정 상의 작업(Work) 흐름을 모델링한 것입니다.

- 도면 심볼 → 작업(Work) → 시뮬레이션 컴포넌트 순으로 현실을 추상화합니다.
- 시뮬레이션은 현재 in-memory(SimStore)로만 동작하며, DB 영속성 없이 실행됩니다.

## 용어 요약

| 용어       | 설명                                                    |
|-----------|--------------------------------------------------------|
| 작업(Work)     | 공정의 최소 단위. 하나의 설비(Station/Buffer/Conveyor) 사용 행위. |
| 공정(Process)  | 여러 작업의 순서적 집합. 하나의 제품 생산 흐름.         |
| 공정 단계(ProcessStep) | 공정 내 개별 작업 순서 항목.                  |
| 도면(Plan)     | 공장 바닥의 공간 배치도. 심볼로 설비 위치 표현.         |
| 심볼 오버라이드 | 도면 심볼에 대한 분류·설명·작업 연결 정보.             |
